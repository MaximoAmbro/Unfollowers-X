Por que este enfoque

Diseñado con las mismas restricciones reales que enfrenta cualquier herramienta de automatizacion en X.

01

Overlay full-screen

Una mascara cubre el 100% del viewport. El usuario ve solo la interfaz del script. X scrollea en background sin distracciones ni scroll visible.

02

DOM Scraping robusto

Lee [data-testid="UserCell"] en lugar de hacer llamadas directas a la API interna de X, que usa headers dinámicos imposibles de replicar sin deteccion.

03

Anti-baneo integrado

Delays aleatorios de 35 a 85 segundos entre cada accion. Limite de 22 unfollows por sesion. Simulacion de hover antes de cada clic para imitar comportamiento humano.

04

Control total

Tabla de resultados con checkboxes individuales. Selecciona exactamente quienes dejar de seguir antes de ejecutar. Boton de parada en cualquier momento.

Instalacion

El metodo recomendado es la Consola de DevTools. Los bookmarklets son bloqueados por la politica CSP de X en navegadores modernos.

1
Navega a tu pagina de seguidos

Abre X e ingresa a la pagina de cuentas que sigues:

https://x.com/TU_USERNAME/following
2
Abre la Consola de DevTools
Windows / Linux F12 o Ctrl + Shift + I
macOS Cmd + Option + I

Selecciona la pestana Console.

3
Copia y pega el script

Abre unfollowers-x.js en GitHub. Selecciona todo el contenido, copialo y pegalo en la consola del navegador. Presiona Enter.

4
Usa el overlay

El overlay aparece automaticamente. Espera a que el escaneo termine, revisa la lista, marca las cuentas que deseas dejar de seguir y haz clic en Dejar de seguir seleccionados.

Limitacion de CSP: X aplica Content Security Policy estricta que bloquea la ejecucion de javascript: bookmarklets en Chrome 98+, Firefox 100+ y Safari 15.4+. Este metodo puede no funcionar.
1
Obtener el bookmarklet

Abre unfollowers-x.min.js en GitHub. Copia la linea que empieza con javascript:.

2
Crear el marcador

En tu navegador, crea un nuevo marcador. En el campo URL, pega la linea copiada (comenzando con javascript:).

3
Ejecutar

Navega a https://x.com/TU_USERNAME/following y haz clic en el marcador. Si la CSP lo bloquea, usa el metodo de DevTools Console.

Arquitectura tecnica

Por que DOM Scraping con Visual Masking en lugar de llamadas directas a la API de X.

Por que no la API de X

  • Headers dinamicos rotativos: X autentica sus endpoints internos con tokens y headers que cambian por peticion. Son imposibles de replicar desde JavaScript del navegador sin ser detectado.
  • CSP bloquea fetch directo: La politica Content Security Policy de X impide hacer fetch() a sus APIs desde scripts inyectados, incluyendo bookmarklets.
  • Rate limits agresivos: La API oficial de X tiene limites muy estrictos y requiere aprobacion de desarrollador. Cualquier automatizacion sin credenciales oficiales es bloqueada.

Por que DOM Scraping con overlay

  • El DOM ya esta autenticado: X renderiza la lista de seguidos en el DOM del navegador del usuario autenticado. Leer ese DOM no requiere ningun header ni token adicional.
  • Overlay como mascara UX: Cubrir el viewport con un position:fixed permite que window.scrollBy() opere en el documento original sin que el usuario vea el scroll frenético.
  • Resistente a cambios de UI: Buscar texto ("Follows you", "Te sigue") en lugar de clases CSS hace el detector mas robusto frente a los frecuentes redeploys del frontend de X.
