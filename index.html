Dos modulos, una herramienta

Cada modulo se habilita automaticamente segun la pagina donde se ejecute el script.

01

Modulo Unfollower

Ejecutar en: x.com/TU_USUARIO/following

  • Escaneo invisible: scroll automatico detras del overlay
  • Detecta todos los no-mutuos sin limite de usuarios
  • Tabla con checkboxes para seleccion granular
  • Delays precisos: 10.23 — 64.32 segundos
  • Cooldown obligatorio cada 10 unfollows: 4-10 minutos
  • Sin limite de unfollows por sesion
02

Modulo Auto-Follow

Ejecutar en: x.com/@USUARIO/followers

  • Carga seguidores en chunks de 250 usuarios
  • Boton "Cargar mas" para chunks adicionales
  • Tabla con checkboxes para seleccion granular
  • Delays: 45 — 95 segundos entre follows
  • Cooldown obligatorio cada 10 follows: 4-10 minutos
  • Maximo 20 follows por lote (regla estricta)
  • Cooldown de 2 horas entre lotes

Sistema anti-baneo sofisticado

Cinco capas de proteccion para imitar comportamiento humano y evitar deteccion por los algoritmos de X.

01

Delays precisos con decimales

Unfollower: 10.23 — 64.32s
Auto-Follow: 45 — 95s
Los decimales rompen la periodicidad estadistica detectada por algoritmos anti-bot.

02

Cooldown obligatorio cada 10 acciones

Pausa aleatoria de 4 a 10 minutos cada 10 unfollows o follows. Simula el comportamiento de un usuario que descansa entre rafagas de actividad.

03

Limite de lote en Auto-Follow

Maximo 20 follows por lote con cooldown de 2 horas entre lotes. Mantiene la actividad dentro de umbrales humanos normales.

04

Simulacion de mouse

Hover con MouseEvent('mouseover') antes de cada clic, con pausa aleatoria entre hover y clic. Los bots hacen clic directo; los humanos, no.

05

Scroll variable entre lecturas

El scroll durante el escaneo usa delays aleatorios de 500ms a 2s para evitar patrones de scroll mecanico regulares.

06

Timeout de sesion

La sesion se detiene automaticamente a las 2 horas. Sesiones excesivamente largas son una senal de alerta para los sistemas de monitoreo.

Por que DOM Scraping y no API GraphQL

Por que NO la API de X

  • Headers dinamicos rotativos: X autentica cada peticion con tokens que cambian por request. Imposibles de replicar desde JS del navegador sin deteccion.
  • CSP bloquea fetch() directo: La politica Content-Security-Policy de X impide llamadas a sus endpoints desde scripts inyectados.
  • API oficial con barreras: Requiere aprobacion de cuenta de desarrollador y tiene rate limits estrictos para acciones de follow/unfollow.

Por que SI DOM Scraping

  • El DOM ya esta autenticado: X renderiza las listas en el navegador del usuario logueado. Leer ese DOM no requiere tokens adicionales.
  • Overlay como mascara UX: position:fixed; inset:0 cubre el viewport mientras window.scrollBy() scrollea el documento original.
  • Busqueda por texto, no por clases: Detectar "Follows you" / "Te sigue" en el innerText es mas robusto que buscar clases CSS que X cambia con cada deploy.

Aviso legal y limitaciones

Este proyecto es de caracter puramente educativo y fue desarrollado como muestra de habilidades tecnicas en JavaScript para un portfolio profesional.

El uso de este script puede violar los Terminos de Servicio de X. Puede resultar en restricciones temporales o permanentes de tu cuenta. El autor no se hace responsable de ninguna consecuencia derivada de su uso.

Limitaciones tecnicas: X actualiza su frontend frecuentemente. Si cambian los data-testid del DOM, el script puede dejar de funcionar hasta ser actualizado. El cooldown de 2 horas entre lotes de Auto-Follow no persiste si se recarga la pagina (no se usa localStorage por diseno).

Usar con moderacion. Esperar al menos 2-3 horas entre sesiones de unfollows y 2 horas entre lotes de follows. No recomendado para cuentas importantes.
